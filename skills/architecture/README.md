# skills/architecture

Skills for shaping the structure of a codebase.

| Skill | Purpose |
|---|---|
| [improve-codebase-architecture/](improve-codebase-architecture/) | Scan a codebase for deepening opportunities and present them as a visual HTML report (Tailwind + Mermaid, before/after diagrams), then grill through whichever candidate you pick. Built on `codebase-design` for vocabulary and `domain-modeling`/`grilling` for the follow-up conversation. |
| [codebase-design/](codebase-design/) | Shared vocabulary for designing deep modules — module, interface, depth, seam, adapter, leverage, locality — plus the dependency-category guide for deepening safely ([DEEPENING.md](codebase-design/DEEPENING.md)) and the parallel-sub-agent pattern for exploring alternative interfaces ([DESIGN-IT-TWICE.md](codebase-design/DESIGN-IT-TWICE.md)). `tdd` and `improve-codebase-architecture` both speak this language rather than each keeping their own copy. |
