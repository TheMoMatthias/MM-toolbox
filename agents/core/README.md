# agents/core

Universally-useful agents — invoke regardless of project domain.

| Agent | When to invoke |
|---|---|
| `code-reviewer.md` | Pre-merge review, codebase health check, architectural review. Focuses on craft + maintainability (function-tester covers correctness). |
| `function-tester.md` | After implementing a new function or refactoring — comprehensive testing for bugs, edge cases, performance, correctness. |
| `systems-architect.md` | High-stakes, correctness-critical, performance-sensitive system design (general). For quant-finance specialization see `agents/quant/quant-trading-architect.md`. |
| `research-engineer.md` | Hypothesis-driven engineering, paper-to-production translation, statistical validation (general). For quant-finance specialization see `agents/quant/quant-researcher.md`. |
| `data-quality-engineer.md` | Pre-training data audit, ETL correctness, statistical validation of calculations (general). For quant-finance specialization see `agents/quant/data-quality-scientist.md`. |
| `ml-engineer.md` | ML system design, training-loop tuning, deployed-model monitoring (general). For quant-finance specialization see `agents/quant/ml-systems-architect.md`. |
